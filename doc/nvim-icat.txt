1. INTRODUCTION                                      *nvim-icat-introduction*

nvim-icat is a Neovim plugin that allows you to display images directly
within Neovim using iTerm2's Inline Images Protocol. Images are displayed in
a dedicated tab with automatic cleanup and intuitive key bindings.

Features:
  - Display images inline using iTerm2's protocol
  - Opens images in a clean, dedicated tab
  - Auto-close with q or <Esc>
  - Support for custom image dimensions
  - Popup sizing from PNG, GIF, JPEG, and WebP headers
  - Works with local files and URLs
  - Debug logging support

==============================================================================
2. REQUIREMENTS                                      *nvim-icat-requirements*

- Neovim 0.5 or higher
- iTerm2 (macOS) with inline image support enabled
- base64 command-line utility (usually pre-installed on macOS)
- curl (required for URL support)
- folke/snacks.nvim (optional, required for |:IcatShowPop|)
- ImageMagick identify or magick (optional fallback for image dimensions
  outside PNG, GIF, JPEG, and WebP)

3. INSTALLATION                                      *nvim-icat-installation*

Using lazy.nvim: >lua
    return {
        'vito-c/nvim-icat',
        config = function()
            require('nvim-icat').setup()
        end,
    }
<

With |:IcatShowPop| support (requires folke/snacks.nvim): >lua
    return {
        'vito-c/nvim-icat',
        dependencies = { 'folke/snacks.nvim' },
        config = function()
            require('nvim-icat').setup()
        end,
    }
<

Using packer.nvim: >lua
    use {
        'vito-c/nvim-icat',
        requires = { 'folke/snacks.nvim' },  -- optional, for :IcatShowPop
        config = function()
            require('nvim-icat').setup()
        end
    }
<

Using vim-plug: >vim
    Plug 'folke/snacks.nvim'  " optional, for :IcatShowPop
    Plug 'vito-c/nvim-icat'

    lua << EOF
    require('nvim-icat').setup()
    EOF
<

5. COMMANDS                                              *nvim-icat-commands*

                                                               *:IcatShow*
:IcatShow {path}
    Display the image at the specified path in a new tab.

    The path can be absolute or relative. Tab completion is available.

    Examples: >
        :IcatShow ~/Pictures/photo.jpg
        :IcatShow ./screenshot.png
<

                                                            *:IcatShowPop*
:IcatShowPop {path}
    Display the image in a floating popup with a rounded border.
    Requires folke/snacks.nvim.

    Examples: >
        :IcatShowPop ~/Pictures/photo.jpg
        :IcatShowPop ./screenshot.png
<

                                                               *:IcatTty*
:IcatTty
    Show TTY diagnostics for image rendering, including the render-time imgcat
    TTY, Neovim process TTY, startup TTY cache, configured fallback TTY, TERM,
    TMUX, current buffer, buffer type, filetype, and window id.

6. FUNCTIONS                                            *nvim-icat-functions*

nvim-icat.setup({opts})                             *nvim-icat.setup()*
    Setup function to initialize the plugin.

    Parameters: ~
        {opts}  (table) Optional configuration table with these fields:
                - imgcat_path: Custom path to imgcat.lua script
                - file_browser: Configure oil.nvim/netrw <CR> integration

    Usage: >lua
        require('nvim-icat').setup()
<

nvim-icat.show_image({image_path}, {options})      *nvim-icat.show_image()*
    Display an image in a new tab.

    Parameters: ~
        {image_path}  (string) Path to the image file
        {options}     (table) Optional table with these fields:
                      - width: Image width (string, e.g., "80", "100px", "50%")
                      - height: Image height (string, e.g., "40", "200px", "80%")

    Usage: >lua
        -- Simple usage
        require('nvim-icat').show_image('/path/to/image.png')

        -- With custom dimensions
        require('nvim-icat').show_image('/path/to/image.png', {
            width = '80',
            height = '40'
        })
<

7. KEY BINDINGS                                      *nvim-icat-keybindings*

When viewing an image, the following key bindings are available:

    q           Close the image viewer tab
    <Esc>       Close the image viewer tab

The image tab will also automatically close when you switch to another tab
(TabLeave event).

==============================================================================
8. CLI POPUP                                                *nvim-icat-cli*

The bin/icat-popup script opens an image popup in an already-running Neovim
instance by sending a remote expression to the Neovim server.

Use an explicit server address: >bash
    bin/icat-popup --server "$NVIM_LISTEN_ADDRESS" ./image.png
<

If NVIM_LISTEN_ADDRESS is set in the environment, --server can be omitted: >bash
    bin/icat-popup ./image.png
<

The script does not render the image from the shell process. It asks the
running Neovim instance to call |:IcatShowPop|, so TTY selection stays inside
the plugin.

==============================================================================
9. POPUP SIZING                                      *nvim-icat-popup-sizing*

|:IcatShowPop| sizes the popup from image metadata when possible. nvim-icat
first reads small file headers directly in Lua for PNG, GIF, JPEG, and WebP.
This does not require external tools and works well when Neovim is running on a
remote host over SSH.

If the header reader cannot determine image dimensions, nvim-icat falls back to:

    1. sips (macOS)
    2. ImageMagick identify
    3. ImageMagick magick identify
    4. editor-size fallback

When the popup notification says "image: ? x ? cells", nvim-icat could not
determine the image dimensions and used the editor-size fallback.

10. CONFIGURATION                                   *nvim-icat-configuration*

nvim-icat can be configured by passing options to the setup function.

Available options:~

imgcat_path                                           *nvim-icat-imgcat_path*
    Type: string
    Default: (auto-detected relative to plugin)

    Custom path to the imgcat.lua script if you want to use a different
    location.

    Example: >lua
        require('nvim-icat').setup({
            imgcat_path = '~/.config/nvim/scripts/imgcat.lua'
        })
<

file_browser                                       *nvim-icat-file_browser*
    Type: table
    Default: { enabled = true, key = "<CR>" }

    Adds a buffer-local mapping in oil.nvim and netrw buffers. When the cursor
    is on a local image file, the mapping opens |:IcatShowPop|. Directories and
    non-image files fall back to the file browser's normal open action.

    Example: >lua
        require('nvim-icat').setup({
            file_browser = {
                enabled = true,
                key = '<CR>',
            },
        })
<

    Disable the file browser integration: >lua
        require('nvim-icat').setup({
            file_browser = {
                enabled = false,
            },
        })
<

11. EXAMPLES                                             *nvim-icat-examples*

Display an image using the command:
>
    :IcatShow ~/Pictures/vacation.jpg
<

Display an image using Lua:
>lua
    :lua require('nvim-icat').show_image(vim.fn.expand('~/Pictures/vacation.jpg'))
<

Display an image with custom size:
>lua
    :lua require('nvim-icat').show_image('screenshot.png', {
        width = '100',
        height = '50'
    })
<

Create a custom key binding:
>lua
    vim.keymap.set('n', '<leader>ii', function()
        vim.ui.input({
            prompt = 'Image path: ',
            completion = 'file'
        }, function(input)
            if input then
                require('nvim-icat').show_image(input)
            end
        end)
    end, { desc = 'Show image' })
<

12. DEBUG LOGGING                                          *nvim-icat-debug*

To enable debug logging, set the IMGCAT_DEBUG environment variable before
starting Neovim:

>bash
    export IMGCAT_DEBUG=1
    nvim
<

Debug logs will be written to:
    - debug.txt          - imgcat script logs
    - debug-plugin.txt   - plugin logs

This is useful for troubleshooting issues with image display.

13. TROUBLESHOOTING                              *nvim-icat-troubleshooting*

Images not displaying~

1. Verify you're using iTerm2 (this plugin only works in iTerm2)
2. Check that inline images are enabled in iTerm2 preferences:
   Preferences > General > Selection > "Copy to pasteboard on selection"
3. Run |:IcatTty| to inspect render-time and startup TTY values
4. Enable debug logging to see detailed information
5. Ensure the image file exists and is readable
6. Install ImageMagick if popup notifications show "image: ? x ? cells" for a
   format outside PNG, GIF, JPEG, and WebP
7. Check that you're not in a nested tmux session (may cause issues)

Command :IcatShow not found~

Make sure you've called the setup function in your Neovim config:
>lua
    require('nvim-icat').setup()
<

Images appear corrupted or garbled~

1. Try using legacy mode by modifying the imgcat.lua script
2. Check file permissions on the image
3. Verify the image file is not corrupted

Performance issues with large images~

Large images may take time to encode and display. Consider:
1. Using smaller image dimensions
2. Resizing images before displaying them
3. Using compressed formats (JPEG vs PNG for photos)
